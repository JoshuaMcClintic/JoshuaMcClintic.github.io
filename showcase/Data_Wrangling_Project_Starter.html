# Defining a function to easily write to or read from a csv file regardless of operating system, as this project was started on a Windows
# system, but was later moved to a Linux system
def to_from_csv(filename, filepath='Raw Data', data=None, to=False, index=False):
    """
    ACCEPTS: filename - Name of the file to write or read
             data - Initially set to None; pandas dataframe to write to a csv file if 'to' is set to True
             to - Initially set to False; if True, the function writes the pandas dataframe in 'data' to a csv file with 'filename' as the name
             index - Initially set to False; used in pandas .to_csv() function as index argument
             
    RETURNS: If 'to' is True, returns True after writing data to csv; if 'to' is False, returns pandas dataframe from csv file
    """
    if to:
        if platform.system() == 'Windows':
            data.to_csv(f'..\\{filepath}\\{filename}.csv', index=index)
        else:
            data.to_csv(f'../{filepath}/{filename}.csv', index=index)
        return True
        
    else:
        if platform.system() == 'Windows':
            df = pd.read_csv(f'..\\{filepath}\\{filename}.csv')
        else:
            df = pd.read_csv(f'../{filepath}/{filename}.csv')
        return df

El Nino Southern Oscillation Index (ENSO)¶

Type: CSV File

Method: Downloaded .zip file from https://data.mfe.govt.nz/table/115303-el-nino-southern-oscillation-index-2016-2020/ and unzipped to "Raw Data" folder

Dataset variables:

  • year
  • month
  • soi: Southern Oscillation Index (SOI) monthly state
  • soi_3mth_mean: Three month rolling mean (right aligned) of SOI monthly state
  • soi_phase: Flags where SOI three-month rolling mean is greater than or equal to 1 (La Niña), less than or equal to -1 (El Niño) or between -1 and 1 (Neutral)
  • soi_simple: soi_phase without tilde

SOI measures the difference between the standardised surface air pressures at Tahiti and Darwin (Australia), and represents the strength of the tropical trade winds. A negative SOI represents below normal air pressure at Tahiti and/or above normal air pressure at Darwin, and weaker than normal trade winds. This dataset contains information on which phase the ocean is in, which is important to determine correltion between El Nino and global temps.

Relative Oceanic Niño Index (RONI)¶

Type: HTML table

Method: BeautifulSoup web scraping from https://www.cpc.ncep.noaa.gov/products/analysis_monitoring/enso/roni/

Dataset variables:

  • Year
  • 3-month period: The three month period for which the mean relative temperature was calculated
  • Mean Relative Temperature: 3 month running mean of ERSST.v5 SST anomalies in the Niño 3.4 region (5°N–5°S, 120°–170°W) with average tropical mean (20°N–20°S) SST anomalies subtracted. The difference is then adjusted so the variance equals the original Niño 3.4 index

The table on the website uses the 3-month period as the columns and year as the rows, with each data point representing the average temperature for the three-month period for that year relative to "A global averaged offset (0.12°C; 1990–2010)" (Huang et al. 2017, J. Climate). This dataset contains temperature information that the ENSO dataset does not.

This dataset is contained in an html table, which needs to be parsed. However, once parsed, the data is stored as a pandas dataframe, which is then exported to a csv file locally, so the code used to initially create the dataframe from the html file is commented out.

I want to clarify that the method used to create the pandas dataframe is almost certainly not the best or most efficient. However, it is probably one of if not the easiest methods to conceptualize.

  1. Create a list to use as the column headers
  2. Create a list containing each value from the table
  3. For each value in the column headers list, enumerate over the values list; if the remainder of the index of the value is equal to the index of the header, add the value to a new list
  4. Create a python dictionary with the header as the key and the new list as the value
  5. Finally, convert the dictionary to a pandas dataframe

Again, there is almost certainly a better method, but this was the first one I thought of. Again, because the dataframe is later saved as a csv file locally, these cells are commented out.
